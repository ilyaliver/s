# s

Gacha Simulator — браузерная интерактивная мини-игра «Поиск сигнальных прыжков» в стиле gacha/симулятора.

<img width="1426" height="733" alt="image" src="https://github.com/user-attachments/assets/ed57be8e-e15b-4ccb-95f4-0bc9b0ad0f78" />

## Кратко о проекте

- Интерактивный интерфейс с космической тематикой.
- Система «поиска» с pity-ограничениями и гарантированными результатами.
- Инвентарь полученных предметов и магазин для внутриигровой валюты.
- Фоновые эффекты и звук (mp3) — аудиофайлы лежат в папке `mp3/`.
- Видео-анимации результатов поиска (mp4) с разными вариантами редкости.
- Сохранение прогресса в локальном хранилище браузера (`localStorage`).

## Структура репозитория

- `index.html` — основной интерфейс и логика приложения.
- `/mp3/` — фоновые аудиотреки и звуковые эффекты.
- `/effect/` — видео-анимации и звуки эффектов для результатов поиска:
  - `normal_x1.mp4` / `normal_x10.mp4` — стандартные результаты
  - `standart_x1.mp4` — альтернативный вариант анимации
  - `garant_x1.mp4` / `garant_x10.mp4` — гарантированные результаты
  - Звуковые файлы (`.mp3`) для каждого типа результата
- `/image/` — графические ресурсы и изображения (иконки и ассеты).
- `/test/` — тестовые версии и вспомогательные файлы.
- `LICENSE` — лицензия проекта (Apache License 2.0).
- `CNAME` — (при наличии) файл для настройки GitHub Pages с кастомным доменом.
- `.gitignore` — список игнорируемых файлов.
- `.github/` — конфигурация GitHub Actions для развертывания.
- `README.md` — этот файл.

> Примечание: содержимое папок `image/`, `effect/` и `test/` может меняться по ходу разработки — в репозитории находятся ассеты и вспомогательные файлы для игры.

## Как открыть

Проект можно открыть напрямую в браузере:

- Откройте файл `index.html` локально в браузере;
- Или разверните репозиторий на GitHub Pages / локальном HTTP-сервере и откройте сайт.

### Локальный запуск

Если нужен локальный сервер, запустите в корне репозитория:

```bash
python3 -m http.server 8000
```

После этого откройте в браузере:

```
http://localhost:8000
```

## Развертывание

- Можно публиковать ветку `main` на GitHub Pages; при использовании `CNAME` репозиторий настроен на кастомный домен (см. файл `CNAME`).
- Статическое развертывание выполняется автоматически через GitHub Actions (см. `.github/workflows/`).

## Примечания

- Для корректной работы используйте современный браузер (последние версии Chrome, Firefox, Edge, Safari).
- Автовоспроизведение фоновой музыки может блокироваться настройками браузера — потребуется разрешение/взаимодействие пользователя.
- Видео-эффекты в папке `/effect/` требуют современного браузера с поддержкой HTML5 Video (mp4 формат).
- Прогресс игры сохраняется в `localStorage` и не синхронизируется между устройствами.
